# render.py 重构计划

基于 commit 3778c29 的代码审查。

## 问题清单

### P0 - 严重影响可维护性（必须修复）

| 问题 | 位置 | 影响 |
|------|------|------|
| 超长函数 generate_disk_texture | 788-1082 (295行) | 职责过多，难以测试和维护 |
| 超长方法 _compile_kernels | 1171-1744 (573行) | 嵌套过深，难以理解 |
| 文档字符串位置错误 | 1943-1960 | render_video 的 docstring 在函数体中间 |
| 双线性插值重复 5 次 | 405-447, 496-531, 1287-1310, 1313-1340, 1343-1378 | 维护成本高，容易出现不一致 |
| RK4 积分重复 3 次 | 1461-1497 | 主光线和两个微分方向完全重复 |
| Taichi 初始化混乱 | 687, 740, 1124 | 多处独立 ti.init() + ti.reset()，可能冲突 |

### P1 - 影响可读性和用户体验（应该修复）

| 问题 | 位置 | 影响 |
|------|------|------|
| CLI 参数命名不直观 | --ar1, --ar2, --pov | 用户难以理解 |
| 参数缺少验证 | parse_args | 可能导致运行时错误 |
| 函数内 import | 1962-1964, 2046 | 违反 PEP8，影响性能 |
| 魔法数字散落 | 全文 | 难以调整和理解物理意义 |
| 变量名太短 | h, w, pw, ph, R | 可读性差 |
| 未使用的函数 | 111-163 | make_all_rays 定义但未调用 |
| 过时注释 | 10-12 | 提到已删除的 NumPy 路径 |

### P2 - 改善代码质量（可选优化）

| 问题 | 位置 | 影响 |
|------|------|------|
| 批量计算模式重复 | 908-941, 969-1002 | 可以抽象为通用函数 |
| Taichi 加速函数重复 | 682-732, 735-785 | 可以合并 |
| 缺少类型注解 | 全文 | IDE 支持不足 |
| 缺少错误处理 | 文件操作、Taichi 初始化 | 鲁棒性差 |
| 全局常量过多 | 30-68 | 应该用配置类管理 |

## 修复计划

### Phase 1: P0 修复

```
P0-1: 文档字符串位置错误 → 移到函数开头
P0-2: RK4 积分重复 → 提取为 @ti.func
P0-3: Taichi 初始化混乱 → 统一在 TaichiRenderer.__init__ 管理，移除子函数中的 ti.init/reset
P0-4: 双线性插值重复 → 提取为 @ti.func
P0-5: generate_disk_texture 拆分 → 拆为多个子函数
P0-6: _compile_kernels 拆分 → 将嵌套 kernel 提取为独立方法
```

### Phase 2: P1 修复

```
P1-1: CLI 参数重命名 → --ar1/--ar2 改为 --disk_inner_radius/--disk_outer_radius
P1-2: 添加参数验证 → fov 范围、ar1 < ar2 等
P1-3: 移动 import 到文件顶部
P1-4: 提取魔法数字为常量
P1-5: 删除未使用的函数 make_all_rays
P1-6: 更新过时注释
```

### Phase 3: P2 优化（可选）

```
P2-1: 合并 _generate_arcs_taichi 和 _generate_hotspots_taichi
P2-2: 添加类型注解
P2-3: 添加错误处理
```

## 执行原则

1. 每次只修复一个问题，提交后再进行下一个
2. 每次修改后运行测试确保功能正常
3. 重构过程中不添加新功能
4. 保持向后兼容，CLI 参数支持旧名称作为别名
