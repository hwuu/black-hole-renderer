## 修复计划

### Phase 1: P0 修复

```
P0-1: 文档字符串位置错误 → 移到函数开头
P0-2: RK4 积分重复 → 提取为 @ti.func
P0-3: Taichi 初始化混乱 → 统一在 TaichiRenderer.__init__ 管理，移除子函数中的 ti.init/reset
P0-4: 双线性插值重复 → 提取为 @ti.func（Taichi 内）和公共函数（NumPy 版，若保留）
P0-5: generate_disk_texture 拆分 → 拆为多个子函数
P0-6: _compile_kernels 拆分 → 将嵌套 kernel 提取为独立方法
P0-7: mipmap 填充性能 → ✅ 已完成（56d7cd6）
```

### Phase 2: P1 修复

```
P1-1: CLI 参数重命名 → --ar1/--ar2 改为 --disk_inner_radius/--disk_outer_radius
P1-2: 添加参数验证 → fov 范围、ar1 < ar2 等
P1-3: 移动 import 到文件顶部
P1-4: 提取魔法数字为常量
P1-5: 删除未使用的函数 make_all_rays
P1-6: 更新过时注释
P1-7: 删除死代码 → _generate_arcs_taichi, _generate_hotspots_taichi, sample_disk_texture
```

### Phase 3: P2 优化（可选）

```
P2-1: 提取 Bloom 模糊为通用函数 → 参数化轴方向
P2-2: 添加类型注解
P2-3: 添加错误处理
```

## 执行原则

1. 每次只修复一个问题，提交后再进行下一个
2. 每次修改后运行测试确保功能正常
3. 重构过程中不添加新功能
4. 保持向后兼容，CLI 参数支持旧名称作为别名
