## 1. 背景与目标

### 1.1 背景

当前吸积盘实现以**赤道面 + 2D 纹理 + 多次穿越累加**为主，已经具备以下能力：

- 程序化纹理生成
- 多次穿越盘面的 ghost image 累加
- 温度/密度启发式结构
- 视频模式下的动态旋转

这套方案在“有吸积盘”层面已经可用，但从“盘体本身的真实感”来看，仍有几个明显限制：

1. **零厚度**：所有结构都贴在单一平面上，缺少盘体感、自遮挡和边缘厚重感。
2. **纹理感较强**：细节更像“会转动的程序贴图”，而不是“被差动旋转持续拉伸的流体结构”。
3. **层次不足**：中面、表层、内缘、外缘的视觉角色不够分明。
4. **动画表达受限**：虽然已有旋转，但本质仍偏向 UV/相位层面的纹理运动，而非统一角速度场驱动下的流体平流。

因此，需要一条与当前实现**平行存在**的吸积盘 V2 路线，重点提升吸积盘本体的几何与材质真实感，而不是继续在 V1 贴图链路上做局部修补。

### 1.2 目标

| 目标 | 说明 |
|------|------|
| **盘体拟真** | 从零厚度平面升级为有限厚度盘体，具备垂向层次与体感 |
| **统一动力学表达** | 所有结构都由同一套 `Ω(r)` 角速度场驱动，真实体现差动旋转 |
| **结构自然** | 纹理从“图案”转向“流体风格结构调制”，弱化规则贴图感 |
| **可独立验证** | 先做与现有 V1 平行的 reference 实现，便于单独调参与验收 |
| **可渐进接入** | 在效果确认后，再以最小风险接入主渲染器 |

### 1.3 非目标

- **不改黑洞测地线积分器**：V2 只聚焦吸积盘本体建模，不重写 Schwarzschild 光线方程。
- **不在第一阶段处理完整相对论光学**：不把 Doppler beaming、引力红移、偏振、康普顿散射作为 V2 的首要目标。
- **不做 Kerr / GRMHD 科研级模拟**：不引入自旋黑洞度规，不直接求解磁流体方程。
- **不直接替换 V1**：先并行实现、独立验证，确认质量后再决定是否接入主路径。
- **不做全 3D 通用体渲染器**：V2 采用 2.5D 有限厚度盘体模型，而不是无约束三维烟雾系统。

### 1.4 与 V1 的关系

V2 与现有 V1 的关系如下：

| 项目 | V1 | V2 |
|------|----|----|
| 盘面几何 | 赤道面薄盘 | 有限厚度盘体 |
| 结构载体 | 2D 极坐标纹理 | 基础物理场 + 结构调制 |
| 动画方式 | 纹理旋转 / 重生成 / 关键帧 | 统一 `φ_adv = φ - Ω(r)t` 平流 |
| 成像方式 | 单次/多次平面命中累加 | 盘体内有限步长发射-吸收积分 |
| 实现路径 | 已在 `render.py` 中 | 计划独立模块化实现 |

V2 不复用 V1 的纹理生成逻辑，不在现有 `generate_disk_texture*` 系列函数上继续叠补，而是单独建立一套盘体模型与采样器。

## 2. 设计总览

### 2.1 为什么需要 V2

吸积盘的真实感，核心不在于“图案更复杂”，而在于以下四件事是否成立：

1. **盘有厚度**，不是一张发光贴纸。
2. **结构会被剪切**，不是整张图刚体转动。
3. **内外层速度不同**，不是所有半径一把梭地同速旋转。
4. **亮度来自体内分布**，不是单层纹理直接出色。

如果继续沿用 V1 的“平面贴图 + 局部修补”路线，能够继续提升局部细节，但很难根本解决“贴图感”和“缺少盘体感”的问题。因此 V2 的重点不是“更复杂的贴图”，而是“重新定义盘本身”。

### 2.2 V2 的核心思想

V2 采用一套**2.5D 有限厚度盘体模型**，由四层组成：

1. **Geometry Layer**：定义盘的局部坐标、内外半径、厚度函数 `H(r)`。
2. **Physical Fields Layer**：定义密度 `ρ(r, z)`、温度 `T(r, z)`、角速度 `Ω(r)`。
3. **Structure Modulations Layer**：在 `(log r, φ_adv)` 空间中定义弱模态、剪切噪声和热斑等调制。
4. **Imaging Layer**：沿穿过盘体的路径做简化发射-吸收积分，得到最终颜色与透明度。

整体流程如下：

```
+----------------+    +----------------+    +----------------+
| Geometry Layer | -> | Physical Fields| -> | Phase Advection|
+----------------+    +----------------+    +----------------+
                                                   |
                                                   v
+----------------+    +----------------+    +----------------+
| Color Mapping  | <- | Path Integrate | <- | Structure Mods |
+----------------+    +----------------+    +----------------+
```

### 2.3 视觉验收标准

V2 至少应满足以下直观验收标准：

- **侧视时更厚重**：高倾角观察下，盘边缘明显比 V1 更有体积感。
- **内区更活跃**：内圈结构旋转更快、剪切更强，外圈更稳定。
- **结构不是均匀转盘**：同一条纹理结构随时间被拉伸、扭曲，而不是整体刚体转动。
- **内缘更可信**：ISCO 附近存在清晰的内缘截断和更复杂的细节层次。
- **颜色与亮度分工明确**：温度决定主色相，密度/路径长度决定亮度与厚重感。

## 3. 盘体模型

### 3.1 几何模型

V2 将吸积盘定义为一个带倾角的局部盘体坐标系：

- 局部原点：黑洞中心
- 法向量：`disk_normal`
- 局部半径：`r = sqrt(x'^2 + y'^2)`
- 局部高度：`z = z'`

盘体并非严格等厚，而是使用半厚度函数 `H(r)`：

```
H(r) = h0 · r · (r / r_in)^β
```

其中：

- `h0`：整体厚度比例，控制“薄盘”还是“稍厚盘”
- `β`：厚度随半径变化的指数，建议较小（如 `0 ~ 0.15`）
- `r_in`、`r_out`：盘体内外边界

第一阶段采用“薄盘到稍厚盘”的范围，不做夸张甜甜圈结构。这样既能获得足够的盘体感，也能避免过度偏离当前项目的视觉语境。

为了让“几何表面”和“基础场 support” 的语义保持一致，垂向再定义一个几何权重：

```
W_z(r, z) = 1 - smoothstep(0, 1, |z| / max(H(r), ε))
```

它的作用不是引入新的物理量，而是让基础场在 `|z| = H(r)` 这条几何表面上平滑收口到 `0`，
避免“场值在边界仍明显非零，但几何上已经被硬截断”的不一致。

**边界约定（必须保持统一）**

- `disk_radial_mask()` 与 `disk_volume_mask()` 使用**闭区间 membership**：
  `r = r_in`、`r = r_out`、`|z| = H(r)` 这些点在几何上仍算“属于盘”
- `disk_radial_weight()` 与 `disk_vertical_weight()` 使用**边界收口**语义：
  在上述精确边界点取值为 `0`
- 因此基础物理场遵循：**几何上边界点仍属于盘内，但物理场在边界上取值为 `0`**

这套约定的目的，是把“几何 membership”和“物理场 support” 明确拆开：

- `*_mask` 只回答“这个点几何上算不算盘内”
- `*_weight` 只负责“基础场在边界如何收口”

后续 `model.py`、`integrator.py` 接入时必须沿用这套边界语义，不能再各自发明一套“边界算 inside 还是 outside”的局部规则。

### 3.2 厚度与密度场

垂向密度分布采用高斯形式：

```
ρ(r, z) = ρ_mid(r) · exp(-0.5 · (z / H(r))²) · W_z(r, z)
```

中面密度 `ρ_mid(r)` 由径向剖面和边界窗函数共同决定：

```
ρ_mid(r) = (r / r_in)^(-rho_power) · W_r(r)
W_r(r) = S_in(r) · S_out(r)
```

其中：

- `rho_power`：径向衰减指数，控制内密外疏
- `S_in(r)`：内边界平滑开启函数
- `S_out(r)`：外边界平滑关闭函数

说明：

- 当前 reference 实现把密度归一化常数吸收到单位系统里，因此没有单独暴露 `ρ0` 参数；
- 文档里统一用 `W_r(r)` 表示代码中的径向平滑权重，它等价于 `S_in(r) · S_out(r)`。

推荐使用 `smoothstep` 而不是硬阈值，以避免在采样和动画中出现明显边缘断裂。

### 3.3 温度与发射率

温度剖面采用标准薄盘启发式形式：

```
T_mid(r) = temp_scale · (r / r_in)^(-3/4) · [1 - sqrt(r_in / r)]^(1/4) · W_r(r)
```

说明：

- 在严格零扭矩边界条件下，`T_mid(r_in) = 0`，最大值出现在 `r_in` 稍外侧；
- 这比“最内圈直接最亮”的硬编码更接近经典薄盘模型；
- 当前实现直接使用参数名 `temp_scale`，不再额外引入抽象符号 `T0`；
- 若最终视觉上希望更强调内缘，可在显示映射层做艺术性增强，而不是直接破坏温度定义。

垂向温度可先做简化，不在第一阶段引入复杂辐射转移，只使用弱垂向衰减：

```
T(r, z) = T_mid(r) · [1 - 0.25 · (|z| / max(H(r), ε))] · W_z(r, z)
```

其中 `W_z(r, z)` 负责几何 support 的平滑关闭；前面的线性项只负责表达“中面更热、表层略冷”的垂向趋势。

说明：

- 当前实现把垂向温度衰减系数固定为 `0.25`，还没有把它单独参数化成 `qT`；
- 如果后续发现这一系数需要频繁调节，再考虑提升为显式参数，而不是在基础层过早增加参数面。

发射率定义为：

```
j(r, z, φ, t) = Cj · ρ(r, z)^α · T(r, z)^β · F_struct(r, φ, t)
```

其中：

- `F_struct`：结构调制因子，决定局部亮暗变化
- `α`、`β`：可调指数，用于在“密度主导”和“温度主导”之间平衡视觉效果

### 3.4 结构调制

V2 中的“纹理”不再是一张固定贴图，而是一个随时间平流的结构调制：

```
F_struct = F_mode · F_shear · F_hotspot
```

约定：

- `F_mode`、`F_shear`、`F_hotspot`、`F_struct` 都是**乘性调制因子**；
- 在盘内它们围绕 `1` 波动；
- 在盘外它们返回中性值 `1`，而不是 `0`；
- “哪里有盘”由几何层和基础物理场负责，不由调制层负责。

各部分职责如下：

1. **F_mode（低频弱模态）**
   - 提供极弱的大尺度不对称性
   - 只建议使用 `m = 1`、`m = 2` 的低阶模态
   - 振幅应低，避免做出“稳定螺旋星系”观感

2. **F_shear（中频剪切扰动）**
   - 作为主要结构来源
   - 在 `(log r, φ_adv)` 空间采样多频噪声
   - 重点呈现被差动旋转拉伸后的丝状/絮状结构

3. **F_hotspot（稀疏热斑）**
   - 用于打破过度平滑与过度程序化的观感
   - 数量少、寿命短、分布偏内圈
   - 不应覆盖主结构；视觉上以局部热点为主，但整体仍围绕 `1` 作正负扰动

设计约束：

- 默认不做大而稳定的 2~4 条强螺旋臂；
- 结构主力来自剪切噪声，而不是规则旋臂；
- 热斑只做点缀，不做“满盘烟花”。

### 3.5 差动旋转与统一平流

V2 的核心约束是：**所有动态结构必须使用同一套角速度场驱动**。

角速度采用开普勒型分布：

```
Ω(r) = omega_scale · (r / r_in)^(-3/2)
```

其中 `omega_scale` 就是当前实现中的整体角速度缩放；若用更抽象的物理记号，它对应于这里的 `Ω_in`。

所有需要随时间运动的结构，都通过统一的平流角坐标 `φ_adv` 采样：

```
φ_adv = wrap(φ - Ω(r) · t)
```

这条约束的意义很重要：

1. **内圈天然转得快，外圈天然转得慢**。
2. **所有组件方向约定一致**，避免再次出现“部分结构正转、部分结构反转”的问题。
3. **无需混用 `np.roll`、贴图刚体旋转、局部相位偏移等多套机制**。

V2 明确禁止“一个组件走相位平移，另一个组件走像素滚动”的双体系做法。

## 4. 成像模型

### 4.1 整体流程

V2 的成像不再是“光线命中一次平面就取纹理色”，而是“光线穿过盘体的一小段路径，在这段路径上累积辐射贡献”。

流程如下：

```
+------------+    +-------------+    +----------------+
| World Ray  | -> | Local Frame | -> | Disk Bounds    |
+------------+    +-------------+    +----------------+
                                            |
                                            v
+------------+    +-------------+    +----------------+
| Final Color| <- | Integrate   | <- | Sample Fields  |
+------------+    +-------------+    +----------------+
```

### 4.2 局部坐标与盘体相交

对于任意世界空间光线：

```
p(s) = p0 + s · d
```

先将其变换到盘体局部坐标系，再与一个保守的盘体包围区域求交。为控制实现复杂度，第一阶段采用以下简化：

1. 用 `r_out` 和 `H_max` 构造保守包围体；
2. 求出光线进入与离开包围体的参数区间 `[s_enter, s_exit]`；
3. 仅在该区间内进行有限个采样点积分。

对于每个采样点，只要满足：

```
r_in <= r <= r_out
|z| <= H(r)
```

则视为落在盘体内，参与辐射累积。

### 4.3 有限厚度积分

V2 的一阶目标是做简化的发射-吸收积分：

```
I_out = ∫ j(s) · exp[-τ(s)] ds
τ(s)  = ∫ α(s) ds
```

离散后可写为：

```python
L = 0
tau = 0
for k in samples:
    if disk_volume_mask(sample_k):
        emissivity = j_k
        opacity = alpha_k
        transmittance = exp(-tau)
        L += transmittance * emissivity * ds
        tau += opacity * ds
```

其中：

- `j_k` 来自 `ρ、T、F_struct`
- `alpha_k` 先使用密度相关项近似：`alpha_k = Ca · ρ_k`
- `ds` 为局部采样步长

为了让边缘视角更有厚重感，可以在第一阶段加入一个**掠射角增益**：

```
alpha_eff = alpha_k · [1 + kg · (1 - |dot(d, disk_normal)|)]
```

这不是严格物理推导，而是用于增强“斜视更厚、正视更透”的视觉效果，且实现简单、收益明确。

### 4.4 颜色映射

V2 中建议将“物理场”和“显示映射”分离：

1. 物理层输出：`ρ、T、j、alpha`
2. 显示层输出：RGB

颜色映射建议保留两种模式：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| `scientific` | 更接近黑体启发式颜色 | 物理展示、参数分析 |
| `cinematic` | 内层白黄、外层橙红，更符合直觉 | 默认渲染、演示输出 |

默认建议使用 `cinematic`，但它仍然必须由温度场驱动，而不是直接写死颜色环带。

## 5. 实现方案

### 5.1 平行实现策略

V2 采用“先独立、后接入”的策略：

1. **先做 reference 实现**：用纯 `NumPy` 构建盘体几何、标量场与积分流程。
2. **先验证盘本体**：优先看静态单帧与动画预览是否自然，不急于接入现有 Taichi 主路径。
3. **确认效果后再迁移**：若视觉与接口都稳定，再考虑迁移到 Taichi 或挂入 `render.py`。

这样做的原因：

- 降低对现有主渲染器的干扰；
- 降低“边写边接、最后很难回退”的风险；
- 便于围绕盘体本身做密集调参与快照测试。

#### 当前实现状态（截至 2026-03-08）

**里程碑状态**

### 5.3 参数草案

V2 的第一版参数先控制在少量关键项，避免陷入“参数爆炸”。下面把“当前已实现参数”和“规划中参数”分开写，避免文档里的概念参数和代码参数继续混在一起：

**当前已实现参数**

| 参数 | 说明 | 建议范围 |
|------|------|----------|
| `r_in` | 内半径 | 与现有 `--ar1` 对齐 |
| `r_out` | 外半径 | 与现有 `--ar2` 对齐 |
| `h0` | 厚度比例 | `0.02 ~ 0.12` |
| `beta_h` | 厚度半径指数 | `0.0 ~ 0.15` |
| `edge_softness` | 径向边界平滑宽度比例 | `0.02 ~ 0.15` |
| `rho_power` | 中面密度径向衰减指数 | `0.8 ~ 1.5` |
| `temp_scale` | 中面温度整体缩放 | 正值，小范围可调 |
| `omega_scale` | 整体旋转速度缩放 | 正值，小范围可调 |
| `mode1_strength` / `mode2_strength` | 低频弱模态强度 | 非负，且和小于 `1` |
| `shear_strength` | 中频剪切扰动强度 | 非负，且小于 `1` |
| `shear_components` | 随机剪切分量数 | 正整数，小到中等 |
| `hotspot_strength` | 热斑调制强度 | 非负，且小于 `1` |
| `hotspot_count` | 热斑数量 | 正整数，建议较小 |
| `hotspot_phi_sigma` | 热斑角向宽度 | 正值 |
| `hotspot_logr_sigma` | 热斑对数半径宽度 | 正值 |
| `hotspot_inner_bias` | 热斑向内圈偏置指数 | 大于 `0` |

**规划中参数**

| 参数 | 说明 | 当前状态 |
|------|------|----------|
| `opacity_scale` | 光学厚度缩放 | 规划中，待有限厚度积分接入 |
| `palette_mode` | 颜色映射模式 | 规划中，待调色层接入 |
| `qT` | 垂向温度衰减系数 | 规划中；当前固定为 `0.25` |

参数原则：

- 先保证物理角色清晰，再考虑增加可调项；
- 先做“少参数、强约束”，避免调参时无法判断问题来自哪里；
- 对乘性调制项，优先保证结果始终为正，避免出现负发射率；
- 尽量复用现有半径与倾角概念，减少新旧系统切换成本。

### 5.4 性能预估

本节给出 `disk_v2` 的**工程估算**，用于帮助判断实现顺序、默认参数和目标机器是否匹配。

说明：

- 以下数字是**估算值，不是实测值**；
- 估算基于当前项目已有 CPU 基线：`docs/design.md` 中记录 `1080p 单帧 ~2s（CPU）`；
- 目标机器假设为 **ThinkPad T480s + i7 低压 4C8T CPU**；
- 该类机器在长时间满载下容易发生热降频，因此同一组参数的连续渲染耗时可能波动 **20% ~ 40%**；
- 本节优先用于评估“哪个阶段贵、贵在哪里”，而不是承诺某个绝对耗时。

#### 5.4.1 成本拆分

`disk_v2` 的成本大致从低到高如下：

1. `geometry / physical_fields`
2. `advection`
3. `structure_modulations`
4. `integrator`
5. `integrator + full black-hole tracing`

结论：

- `H(r)`、`ρ(r,z)`、`T(r,z)`、`Ω(r)` 这些基础场本身非常便宜；
- 真正的主要成本来自**结构调制生成**和**有限厚度积分**；
- 一旦接入现有黑洞主渲染路径，总耗时将以“现有光线追踪耗时 + 盘体额外采样耗时”的形式叠加。

#### 5.4.2 分阶段估算

| 里程碑 | 主要内容 | T480s 估算 |
|--------|----------|------------|
| **Phase 1** | `geometry + physical_fields + structure_modulations + preview` | 单测通常 `0.1 ~ 0.5s`；静态结构预览 `1024×256` 约 `40 ~ 120ms`，`2048×512` 约 `120 ~ 350ms` |
| **Phase 2** | 统一平流 `φ_adv` | 每帧通常只比静态结构多 `10 ~ 120ms` |
| **Phase 3** | 有限厚度积分（disk-only preview） | `512×512` 约 `0.08 ~ 0.25s`；`720p` 约 `0.25 ~ 0.8s`；`1080p` 约 `0.6 ~ 2.0s` |
| **Integration** | 接入现有黑洞主渲染器 | 总耗时通常达到 `720p: 2 ~ 6s` / `1080p: 5 ~ 12s` |

上表中的 `Phase 3` 与 `Integration` 默认采用以下前提：

- 已做盘体包围体裁剪；
- 仅对命中盘体区域做局部积分；
- 每像素盘内采样步数控制在 `8 ~ 16`；
- reference 版本主要基于 `NumPy` 向量化，而不是 Python 双重 for 循环。

#### 5.4.3 组件级估算

从实现角度拆分，`disk_v2` 各组件的相对成本大致如下：

| 组件 | 主要操作 | 相对成本 | 说明 |
|------|----------|----------|------|
| `params.py` | 参数校验与基础配置 | 很低 | 基本不会成为瓶颈 |
| `geometry.py` | `smoothstep`、厚度函数、`*_mask`、`*_weight` | 低 | 几乎不会成为瓶颈 |
| `physical_fields.py` | `pow / sqrt / exp` 基础物理场 | 低 | 基础层可放心先做 |
| `structure_modulations.py` | 多尺度噪声、`sin/cos/log`、热斑叠加 | 中 | 是第一个明显耗时点 |
| `advection` | `φ_adv` 坐标变换与重采样 | 低-中 | 动态本身不贵，重建结构才贵 |
| `integrator.py` | 每像素多步采样、`exp(-τ)` 累积 | 高 | 是 `disk_v2` 的主成本中心 |
| 主渲染接线 | 黑洞光线追踪 + 盘体积分 | 很高 | CPU 上总成本最高 |

#### 5.4.4 采样步数对耗时的影响

有限厚度积分阶段，耗时与盘内采样步数近似成正比：

- `8` 步：推荐作为开发默认值；
- `12 ~ 16` 步：适合作为静帧默认值；
- `24 ~ 32` 步：质量会提升，但在 T480s 上通常需要比 `8 ~ 12` 步多出 **1.8 ~ 2.5 倍** 的时间。

因此，V2 的实现不应一开始就以“高步数换质量”为主要策略，而应优先做好：

- 盘体包围体裁剪；
- 仅对盘区域积分；
- 结构调制分辨率和积分分辨率解耦；
- 尽量避免 Python 层逐像素循环。

#### 5.4.5 T480s 友好的默认档位

为了让 V2 在 T480s 这类机器上保持可开发、可预览、可出图，建议预设两档：

| 档位 | 结构调制分辨率 | 输出分辨率 | 积分步数 | 目标 |
|------|--------------|------------|----------|------|
| **开发预览档** | `1024×256` 或 `1536×384` | `720p` | `8` | 快速迭代、看结构和旋转 |
| **静帧输出档** | `2048×512` | `1080p` | `12 ~ 16` | 静态出图、效果确认 |

这两个档位的核心思路是：

- 第 1~3 步优先保证结构和动画表达正确；
- 第 4 步再把额外预算投入到有限厚度积分；
- 在 CPU 机器上，优先控制采样范围和采样步数，而不是一味提高全局分辨率。

## 6. 测试方案

### 6.1 单元测试

V2 需要从一开始配套单元测试，防止后续在调视觉时破坏基础约束。

**已实现：`test_disk_v2_array_utils.py`**

- 标量输入与数组输入统一转为 `float64` 数组
- 数值结果恢复为与原始输入一致的标量或数组形状
- 布尔结果恢复为与原始输入一致的布尔标量或布尔数组

**已实现：`test_disk_v2_physical_fields.py`**

- `H(r) > 0` 且连续
- `Ω(r)` 随半径单调下降
- `ρ_mid(r)` 在内外边界外趋近于 0
- `T_mid(r)` 在 `r_in` 附近起步、稍外侧达到峰值后向外衰减

**已实现：`test_disk_v2_structure_modulations.py`**

- 弱模态调制保持正值且幅度受控
- 剪切纹理调制在相同 `seed` 下可复现、不同 `seed` 下可区分
- 热斑调制在内圈扰动更强，且围绕 `1` 波动
- 总结构调制在盘内有限、为正，盘外返回中性值 `1`

**规划中：`test_disk_v2_advection.py`**

- 同一 `r` 上的结构在 `Δt` 后应推进 `Ω(r) · Δt`
- 内圈相位推进量必须大于外圈
- 所有组件共享同一平流方向，不允许出现符号不一致

**规划中：`test_disk_v2_integrator.py`**

- 更长穿越路径应产生更大的累计 `τ`
- 离盘路径贡献应为 0
- 在相同 `ρ/T` 下，掠射角增益应提升边缘厚重感

### 6.2 图像回归

建议增加固定参数、固定随机种子的快照回归：

- face-on 静态图
- edge-on 静态图
- 3 帧时间切片图（`t0 / t1 / t2`）

图像回归的重点不是像素级完美一致，而是保护以下特征不被无意改坏：

- 内缘截断位置
- 厚度轮廓
- 内外圈旋转速度差异
- 结构拉伸方向

### 6.3 稳定性与性能

V2 的 reference 实现不以极致性能为第一目标，但需要满足：

- 固定 seed 下输出稳定；
- 参数连续变化时结果平滑，不出现突然翻转或断裂；
- 采样步数增加时结果应逐步收敛，而不是剧烈漂移。

在接入主渲染器前，再单独评估是否需要 Taichi 化。

## 7. 实施顺序

### 7.1 Phase 1：静态参考实现

目标：只解决“盘长什么样”。

实现内容：

1. 局部盘体坐标系
2. `H(r)`、`ρ(r,z)`、`T(r,z)`、`Ω(r)`
3. 静态结构调制 `F_struct(r, φ, t=0)`
4. 单帧预览输出

验收点：

- face-on 与 edge-on 都有明显盘体感；
- 不依赖旧纹理代码；
- 不引入动画与主渲染器接线。

### 7.2 Phase 2：时间平流与动画

目标：只解决“盘怎么转”。

实现内容：

1. 引入统一 `φ_adv = φ - Ω(r)t`
2. 所有结构组件统一切换到平流坐标采样
3. 输出 3~5 帧对比图或短预览动画

验收点：

- 内快外慢清晰可见；
- 结构表现为剪切和缠绕，而非刚体转动；
- 不再存在不同组件方向不一致的问题。

### 7.3 Phase 3：有限厚度积分

目标：只解决“为什么它看起来像体积发光物”。

实现内容：

1. 光线与盘体保守包围体求交
2. 盘体内部有限步长采样
3. 发射-吸收累积
4. 掠射角增益

验收点：

- edge-on 时明显比 Phase 1 更厚重；
- 局部自遮挡成立；
- 不会把盘整体涂成一团不透明光带。

### 7.4 Phase 4：接入主渲染器

目标：只解决“如何以最小风险集成”。

实现内容：

1. 设计 `disk_model = v1 / v2` 切换入口
2. 将 V2 sampler 接入现有相机/光线接口
3. 与现有 skybox、黑洞光线追踪路径拼接

验收点：

- V1 仍可用；
- V2 能独立开关；
- 接入后不会影响现有非 V2 渲染路径。

## 8. 风险与取舍

| 风险/取舍 | 说明 | 应对 |
|-----------|------|------|
| **过度复杂化** | 一开始就做 full 3D 容易失控 | 固定为 2.5D 有限厚度盘体 |
| **视觉像星系** | 低频旋臂过强会偏离吸积盘质感 | 默认弱模态，主结构交给剪切噪声 |
| **参数爆炸** | 参数太多会让调试失真 | 第一阶段只开放少量核心参数 |
| **方向约定混乱** | 动画组件各自实现容易符号不一致 | 所有动态统一走 `φ_adv` |
| **过早接入主路径** | 一边接入一边调效果风险高 | 先 reference，后集成 |

V2 的核心取舍是：

- 放弃“最省事的贴图增强”，换取更自然的盘体感；
- 放弃“科研级完整物理”，换取能落地、可验证、能接入当前项目的中间方案；
- 放弃“立刻并入主路径”，换取更低的实现风险和更清晰的测试边界。
